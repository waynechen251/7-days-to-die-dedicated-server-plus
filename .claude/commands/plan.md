# 創建執行計畫

根據用戶需求創建執行計畫文件。

## 輸入
$ARGUMENTS

## 執行步驟

1. **理解需求**: 確認需求範圍，必要時提問
2. **代碼分析**: 讀取相關檔案
3. **方案設計**: 設計符合專案風格的方案
4. **產出計畫**: 創建 `docs/plans/YYYYMMDD-任務名稱.md`

## 分析清單

- [ ] 理解需求完整範圍
- [ ] 找出相關檔案
- [ ] 確認代碼風格
- [ ] 考慮邊界情況
- [ ] 評估影響範圍

## 計畫規範

- 存放: `docs/plans/YYYYMMDD-任務名稱.md`
- 模板: `docs/plans/TEMPLATE.md`
- 必要元素: 需求摘要、影響範圍、執行步驟、測試檢查點

## 代碼風格

**後端**: CommonJS, 2 空格縮排, `log()`/`error()` 函數
**前端**: IIFE 模式, `window.App` 命名空間, `data-i18n` 屬性
