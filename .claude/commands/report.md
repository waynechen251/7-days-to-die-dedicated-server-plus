# 問題回報

在計畫文件中記錄執行問題。

## 輸入
$ARGUMENTS (計畫檔名)

## 回報內容

1. 停止於哪個步驟
2. 問題類型 (代碼錯誤/檔案不存在/行號不符/邏輯衝突/其他)
3. 詳細描述
4. 已嘗試的方式
5. 建議

## 操作

1. 讀取計畫文件
2. 在「執行問題回報」區塊新增記錄
3. 將狀態改為「待處理」
4. 儲存文件

## 格式

#### 問題 N (YYYY-MM-DD HH:MM)
- **停止於步驟**: 步驟 X
- **問題類型**: [類型]
- **問題描述**: [描述]
- **已嘗試**: [嘗試過的方式]
- **建議**: [建議]
